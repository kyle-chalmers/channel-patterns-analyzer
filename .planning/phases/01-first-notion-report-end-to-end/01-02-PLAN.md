<objective>
Create the project-local `write-notion-report` Skill that takes a strict report dictionary from the analyzer and publishes it as a child page on the channel-patterns Notion page. The Skill is the only component that touches Notion — the analyzer (Plan 03) never builds blocks itself.

Purpose: Encapsulates Notion-block-rendering, error categorization, and the parent/child page model so the recipe in Plan 03 stays focused on data orchestration. Lives in the repo so viewers/forkers see how it works.

Output: A single committed `SKILL.md` (~150 lines) whose frontmatter and body are precise enough that Claude reading them in isolation reliably invokes the Skill when handed an analyzer report dict.
</objective>
