## Plan Structure Rationale

This plan owns **Slice 3 of MVP Phase 3 (Runbook coverage)**. It depends on Plan 03-03 because the Run-now checklist in `docs/schedule.md` links into specific runbook section anchors that this plan creates, AND because cloud-specific failure modes only became real after Plan 03-03 wired the cloud-routine setup walkthrough.

This is a docs-only plan. No code is changed in scripts or SQL. The one recipe edit is a single-sentence closing instruction in the failure-handling step (anchoring ERR-03). The analyzer's BigQuery-mode and CSV-mode runtime behavior is identical before and after this plan.

**Wave 3 placement.** Strictly depends on Plan 03-03 (the schedule.md Run-now checklist anchors that this plan creates).

**MVP slice check.** After this plan ships, the analyzer is end-to-end runnable in three modes (CSV, BigQuery local, BigQuery cloud routine) AND every failure path leads to a named runbook section. The phase is complete; ROADMAP can transition.

**Tech-debt fold-ins** per `03-RESEARCH.md` § "Tech Debt Scope Decision":
- Fix the `BUSINESS_RULES.md §6` reference rot in `docs/maintenance.md:9` (Phase 1 Plan 01-04 fixed the runbook side; this is the remaining maintenance.md side).
- Fix the "Fix (scheduled cloud routine)" paragraph in `docs/runbook.md:23` which currently names service-account keys (conflicts with D-01).

## Phase Goal (User Story)

**As a** solo channel operator hitting an unfamiliar failure mode at 9:01 AM on a Monday, **I want to** scan `docs/runbook.md` for a section heading matching the error category in `summary.json` or the Anthropic UI's transcript and read a Symptom / Fix / Recording block, **so that** I recover the run without having to remember internal recipe steps or read SQL files cold.

<objective>
Expand `docs/runbook.md` from 6 sections to ~13 sections so every error category the recipe emits AND every cloud-specific failure mode `03-RESEARCH.md` inventoried has a named section with the Symptom / Fix / Recording template. Fix two pre-existing portability bugs: the "service account key" paragraph in `docs/runbook.md:23` (conflicts with D-01) and the `BUSINESS_RULES.md §6` reference in `docs/maintenance.md:9` (cross-reference rot). Add a one-line closing instruction to the recipe's failure-handling step anchoring ERR-03 ("If this failure mode is not in docs/runbook.md, add it as part of the fix").

Purpose: per **ERR-01** every failure mode hit during Phase 1+2 builds AND the cloud-specific modes `03-RESEARCH.md` predicts (which Phase 3's smoke test will surface) must have a named runbook section with recovery steps. Per **ERR-03** the discipline of "newly-encountered failure modes get added to docs/runbook.md + CHANGELOG.md as part of the fix" needs to be visible from the recipe itself, not just in `docs/maintenance.md`.

Output: a runbook that an operator hitting any recipe error category or cloud-routine failure can scan in under 60 seconds and find the matching recovery section.
</objective>
